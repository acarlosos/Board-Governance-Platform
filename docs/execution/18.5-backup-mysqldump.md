# Fase 18.5 — Backup nocturno via `mysqldump` (substitui Spatie)

> Spec criada para destravar `PO1` da revisão de readiness. **Substitui** a alternativa Spatie (incompatível com Laravel 13). Mantém-se o escopo MVP: backup diário, retenção simples, sem S3, sem encriptação, sem multi-destination.

## 1. Contexto

O pacote `spatie/laravel-backup` ainda não tem release compatível com Laravel 13 (exige `illuminate/console ^10-12`). O caminho aprovado pelo Owner é **`mysqldump` nativo + `gzip` + retenção simples** via Artisan commands agendados. S3, encriptação, alertas Slack e backup de storage ficam para pós-MVP.

## 2. Decisões finais confirmadas

| ID | Decisão |
|---|---|
| **D45** | Backup = `mysqldump` em pipe para `gzip`. Sem dependência externa. Único disco: `local`. |
| **D46** | Localização: `storage/app/backups/`. Gitignorada. Permissions `chmod 750`. |
| **D47** | Filename: `bgp-{env}-{Y-m-d-His}.sql.gz`. |
| **D48** | Retenção configurável via `config/backup.php` + env `BACKUP_RETENTION_DAYS` (default `14`). |
| **D49** | Schedule em `bootstrap/app.php`: `backup:run` diário 03:00 + `backup:clean` diário 03:30. `withoutOverlapping(60)`. |
| **D50** | **Segurança credenciais:** o `mysqldump` recebe credenciais por **`--defaults-file`** (arquivo temporário 0600 criado pelo command e removido em finally). **Proibido** `-pPASSWORD` na linha de comando (visível em `ps`). |
| **D51** | Sem cobertura de `storage/app/private` (documentos por tenant) nesta fase. Decisão Owner: backup de filesystem fica pós-MVP. **Risco aceitável** documentado em runbook. |
| **D52** | Sem notificação proactiva (email/Slack) nesta fase. Operador inspecciona `storage/logs/laravel.log` + lista `storage/app/backups/` na rotina diária. Notificações entram pós-MVP. |
| **D53** | Restore = procedimento manual documentado em runbook (`gunzip < f.sql.gz | mysql -u … -p …`). Sem comando `backup:restore` nesta fase. |

## 3. Escopo

- Artisan `backup:run` — dump SQL + gzip → `storage/app/backups/bgp-{env}-{ts}.sql.gz`.
- Artisan `backup:clean` — remove ficheiros `bgp-*.sql.gz` em `storage/app/backups/` com `mtime > retention_days`.
- `config/backup.php` com `retention_days`, `disk`, `path` configuráveis.
- Schedule diário em `bootstrap/app.php`.
- Smoke test feature (`tests/Feature/Console/Backup/*Test.php`).
- Documentação operacional actualizada (`deploy.md`, `runbook.md`).
- Item S10 do `security-checklist.md` re-redigido para apontar para `backup:run`.

## 4. Fora do escopo

- Backup de `storage/app/private` (filesystem) — pós-MVP (D51).
- S3 / object storage — pós-MVP.
- Encriptação at-rest do dump — pós-MVP.
- Multi-destination — pós-MVP.
- Alertas Slack/Discord/email — pós-MVP (D52).
- Comando `backup:restore` automatizado — pós-MVP (D53).
- Backup incremental / point-in-time — fora do MVP.
- Reinstalar/avaliar Spatie — só quando houver release Laravel 13.

## 6. Implementação passo a passo

1. **`BackupRunCommand`** — assinatura `backup:run {--only-db : reservado para evolução}`. Construir caminho de saída, garantir `storage/app/backups/` existe com permissions 750. Gerar arquivo temporário `.my.cnf` 0600 com `[client] user=… password=…` lido de `config('database.connections.mysql.*')`. Invocar `mysqldump --defaults-file=… --single-transaction --quick --routines --triggers --no-tablespaces DB | gzip -c > FILE.sql.gz`. Em `finally`, apagar `.my.cnf`. Logar resumo (caminho + tamanho) em `Log::info`. Devolver `0` em sucesso, `1` em qualquer falha (e log warning).
2. **`BackupCleanCommand`** — assinatura `backup:clean`. Listar `storage/app/backups/bgp-*.sql.gz`, comparar `filemtime()` com `now()->subDays(config('backup.retention_days'))`, remover. Logar `Log::info` com contagem (apagados / mantidos).
3. **`config/backup.php`** — chaves `retention_days` (env `BACKUP_RETENTION_DAYS`, default `14`), `disk` (default `local`), `path` (default `backups`).
4. **Schedule + envs** — adicionar ao `bootstrap/app.php → withSchedule`: `$schedule->command('backup:run')->dailyAt('03:00')->withoutOverlapping(60)->onOneServer()->runInBackground();` e o clean às `03:30`. Adicionar `BACKUP_RETENTION_DAYS` a `.env.example` e bloco `BACKUP_*` ao `.env.production.example`.
5. **Tests + docs** — testes: (a) `backup:run` cria ficheiro com prefix esperado e content gzip (mockar `Process::fake()` se necessário, ou usar driver `sqlite` em CI e validar **só** que o command falha gracefully e gera log apropriado — porque `mysqldump` não corre em SQLite); (b) `backup:clean` remove ficheiros antigos e preserva recentes (mock `filemtime` via fixture). Actualizar `deploy.md`, `runbook.md`, `security-checklist.md`. Preencher `result.md`.

> **Nota CI:** suite usa SQLite. `BackupRunCommandTest` deve **mockar** o pipeline `mysqldump` (via `Symfony\Component\Process\Process` injectado ou wrapper service `MysqlDumpRunner` testável) — não chamar `mysqldump` real. `BackupCleanCommand` é puro filesystem e testa-se com fixtures.

## 7. Regras de segurança / tenancy

- **Credenciais DB nunca em CLI argv** (D50). `--defaults-file` é o único caminho aceite.
- Arquivo temporário `.my.cnf` em `storage/framework/tmp/` (ou `sys_get_temp_dir()`), criado com permissions **0600** e removido em `finally` mesmo em exception. Cobrir num teste unitário.
- Ficheiros de backup **não** devem ser servidos publicamente. `storage/app/backups/` está em `storage/app/` (não `public/`), portanto OK por defeito. Confirmar no security-checklist S8.
- Log do command **nunca** inclui password, user ou host completo. Mensagem aceitável: `"Backup written: storage/app/backups/bgp-production-2026-05-13-030001.sql.gz (12.4 MB)"`.
- Sem `tenant_id` no log do command (esta acção é cross-tenant por natureza; opera ao nível DB).
