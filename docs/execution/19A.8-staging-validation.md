1. **Deploy staging**
   - DevOps faz deploy do branch atual (que contém 19A.0–19A.7 + fix Filament + 19B.1–19B.3) em ambiente staging com MySQL real.
   - Confirmar `php artisan filament:upgrade` corre sem erro (regressão do fix `AdminPanelProvider`).
   - Confirmar seed `RolesAndPermissionsSeeder` aplicada (5 perfis).
2. **Validar flag OFF**
   - `.env` staging: `BGP_DASHBOARD_USE_EXECUTIVE_WIDGETS=false` + `php artisan config:clear`.
   - Login como `tenant_admin`: confirmar 6 widgets legacy visíveis e Executive Dashboard widgets ocultos.
   - Conferir `storage/logs/laravel.log` sem `ERROR` durante 30 min de uso.
3. **Validar flag ON — visual V1–V9 + matriz de perfis**
   - `.env` staging: `BGP_DASHBOARD_USE_EXECUTIVE_WIDGETS=true` + `php artisan config:clear`.
   - Executar checklist V1–V9 (ver §9). Capturar screenshot por item.
   - Para cada perfil da tabela canónica (`features/dashboard.md` → "Comportamento por perfil"), confirmar comportamento observado bate certo. Registar diferenças.
4. **Validar performance P1–P6 + multi-tenant**
   - Cold (cache vazio): medir TTFB + queries com Telescope/DevTools. Alvo: TTFB ≤ 1.2 s, queries ≤ 25.
   - Warm (após cold): mesma medição. Alvo: TTFB ≤ 400 ms, queries ≤ 5.
   - Soak 24 h: deixar staging com flag ON e tráfego sintético (script simples ou monitor humano). 0 `ERROR` recorrente.
   - Multi-tenant: criar 2 tenants A e B com dados distintos; login `tenant_admin@A` vê só dados de A; `super_admin` vê agregados globais; Priorities/Activity vazias para super_admin (D4).
5. **Validar rollback + selar findings**
   - `BGP_DASHBOARD_USE_EXECUTIVE_WIDGETS=false` + cache clear → confirmar retorno 6 widgets legacy.
   - Preencher `19A.8-staging-validation.result.md` com tabela de findings, screenshots, JSON medições, decisão GO/NO-GO assinada pelo QA owner.

## 7. Regras de segurança / tenancy

- **Cross-tenant zero tolerance:** se algum widget mostrar dado de outro tenant durante validação, **PARAR e abrir bug crítico**. Não fazer GO.
- Activity feed não pode listar `audit_logs` de outro tenant — verificar manualmente abrindo widget como `tenant_admin@A` após inserir audits em B.
- Priorities não pode mostrar itens omitidos com mensagem "X omitidos" (anti-enumeração D9).
- Nenhum log staging pode imprimir `tenant_id` cru fora de canais `audit` — confirmar `storage/logs/laravel.log` não tem PII.
